yum-2ndquadrant Cookbook
============

The yum-2ndquadrant cookbook takes over management of the default
repositoryids used by 2ndquadrant. It allows attribute manipulation of
`2ndquadrant`.

Requirements
------------
* Chef 11 or higher
* yum cookbook version 3.0.0 or higher

Attributes
----------
The following attributes are set by default

``` ruby
default['yum']['2ndquadrant']['baseurl'] = 'http://2ndquadrant.org/mirrors-2ndquadrant.el6'
default['yum']['2ndquadrant']['baseurl'] = 'ELRepo.org Yum Repository'
default['yum']['2ndquadrant']['gpgkey'] = 'http://2ndquadrant.org/RPM-GPG-KEY-2ndquadrant.org'
default['yum']['2ndquadrant']['enabled'] = true
```

Recipes
-------
* default - Walks through node attributes and feeds a yum_resource
  parameters. The following is an example a resource generated by the
  recipe during compilation.

```ruby
  yum_repository '2ndquadrant' do
    mirrorlist 'http://2ndquadrant.org/mirrors-2ndquadrant.el6'
    description 'ELRepo.org Yum Repository'
    enabled true
    gpgcheck true
    gpgkey 'http://2ndquadrant.org/RPM-GPG-KEY-2ndquadrant.org'
  end
```

Usage Example
-------------
To disable the 2ndquadrant repository through a Role or Environment definition

```
default_attributes(
  :yum => {
    :2ndquadrant => {
      :enabled => {
        false
       }
     }
   }
 )
```

To enable the 2ndquadrant repository with a wrapper cookbook, place
the following in a recipe:

```
node.default['yum']['2ndquadrant']['enabled'] = true
include_recipe 'yum-2ndquadrant'
```

More Examples
-------------
Point the 2ndquadrant repositories at an internally hosted server.

```
node.default['yum']['2ndquadrant']['enabled'] = true
node.default['yum']['2ndquadrant']['baseurl'] = 'https://internal.example.com/2ndquadrant'
node.default['yum']['2ndquadrant']['sslverify'] = false

include_recipe 'yum-2ndquadrant'
```
